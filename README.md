# How to Run the Application

This document provides detailed steps on how to set up, run, and test the `jubelio-project` application.

## Prerequisites

Before running the application, ensure you have the following installed:

- **Node.js** (v16 or later)
- **npm** (Node Package Manager) or **yarn**
- **PostgreSQL** database (if running with a real database)
- **TypeScript** (if not installed globally)

## Setup

1. **Clone the Repository**:
   ```bash
   git clone <repository-url>
   cd jubelio-project
   ```

2. **Install Dependencies**:
   Run the following command to install all required dependencies:
   ```bash
   npm install
   ```

3. **Set Up Environment Variables**:
   Create a `.env` file in the root directory and add the following:
   ```env
   DB_HOST=localhost
   DB_PORT=5432
   DB_USER=your_database_user
   DB_PASSWORD=your_database_password
   DB_NAME=your_database_name
   ```
   Replace the placeholders with your actual database credentials.

## Running the Application

### Development Mode
To run the application in development mode:
```bash
npm run dev
```
This command will use `ts-node` to execute the TypeScript files directly.

### Production Mode
To build and start the application in production:
1. Build the project:
   ```bash
   npm run build
   ```
2. Start the built application:
   ```bash
   npm start
   ```

## Database Migrations

### Run Migrations
To apply the database migrations:
```bash
npm run migrate
```

### Rollback Migrations
To roll back the last applied migration:
```bash
npm run rollback
```

## Running Tests
To run the test suite:
```bash
npm test
```
This will execute all tests using Jest with detailed output.
```

## Project Structure
- `app.ts`: Main entry point for the application.
- `routes/`: Contains route definitions.
- `test/`: Contains test files for the application.
- `migrate.ts`: Script for handling database migrations.
- `dist/`: Contains compiled JavaScript files (created after building the project).

## Notes
- Ensure the PostgreSQL server is running and accessible before running the application.
- Update the `.env` file with the correct configuration for your database.

## Troubleshooting
- **Database Connection Issues**:
  Ensure the credentials in the `.env` file are correct and the PostgreSQL server is running.
- **TypeScript Errors**:
  Run `npm run build` to compile and check for any TypeScript errors.
- **CORS Errors**:
  Update the CORS configuration in the application if needed.

For any additional questions, refer to the official documentation or reach out to the project maintainer.
